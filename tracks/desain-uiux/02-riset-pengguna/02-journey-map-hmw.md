# User Journey Map & How Might We

Setelah memahami pengguna lewat interview, langkah berikutnya adalah memetakan pengalaman mereka secara menyeluruh — dan menemukan peluang desain di dalamnya.

## User Journey Map

Journey map memvisualisasikan seluruh pengalaman pengguna saat berinteraksi dengan produk — dari sebelum menggunakan hingga setelah selesai.

### Komponen Journey Map

```
Persona:  Sandi, 17 tahun, ingin belajar programming

Fase:     | Sadar      | Cari Info   | Daftar      | Belajar     | Selesai     |
----------|------------|-------------|-------------|-------------|-------------|
Tindakan: | Dengar dari| Google      | Isi form    | Buka lesson | Tandai      |
          | teman      | "belajar    | registrasi  | pertama     | selesai     |
          |            | coding SMA" |             |             |             |
----------|------------|-------------|-------------|-------------|-------------|
Pikiran:  | "Menarik,  | "Banyak     | "Formulirnya| "Materinya  | "Akhirnya   |
          | coba lihat"| pilihan,    | panjang     | bagus tapi  | selesai     |
          |            | mana yang   | banget"     | kodenya     | satu lesson"|
          |            | bagus?"     |             | error"      |             |
----------|------------|-------------|-------------|-------------|-------------|
Emosi:    |    😐      |    😕       |    😤       |    😊→😟    |    😊       |
----------|------------|-------------|-------------|-------------|-------------|
Pain:     |            | Terlalu     | Form terlalu| Kode tidak  |             |
          |            | banyak opsi | panjang     | bisa jalan  |             |
----------|------------|-------------|-------------|-------------|-------------|
Peluang:  |            | Halaman     | Sederhanakan| Tambah      | Gamifikasi  |
          |            | perbandingan| onboarding  | "Run Code"  | progress    |
```

### Cara Membuat Journey Map

1. **Tentukan persona** — siapa yang kamu petakan?
2. **Tentukan skenario** — tugas spesifik apa yang dilakukan?
3. **Identifikasi fase** — tahapan besar dalam perjalanan
4. **Isi setiap sel** — tindakan, pikiran, emosi, pain point
5. **Temukan peluang** — di mana pain point terbesar?

## How Might We (HMW)

HMW adalah teknik mengubah masalah menjadi pertanyaan peluang desain.

### Formula

```
"Bagaimana kita bisa [mengatasi masalah] untuk [pengguna] 
 sehingga [hasil yang diinginkan]?"
```

### Contoh Transformasi

| Pain Point | How Might We |
|-----------|-------------|
| "Form registrasi terlalu panjang" | HMW membuat onboarding terasa cepat dan tidak melelahkan? |
| "Kode di lesson tidak bisa dijalankan" | HMW memungkinkan siswa mencoba kode langsung di browser? |
| "Tidak tahu harus mulai dari mana" | HMW membantu siswa menemukan starting point yang tepat untuk mereka? |

### Tips HMW yang Baik

- **Terlalu sempit:** "HMW membuat tombol submit lebih besar?" → terlalu spesifik, tidak membuka kreativitas
- **Terlalu luas:** "HMW membuat belajar menyenangkan?" → terlalu abstrak, sulit dieksekusi
- **Tepat:** "HMW mengurangi friction saat siswa pertama kali membuka lesson?"

## Dari HMW ke Ide

Setelah punya pertanyaan HMW, brainstorm sebanyak mungkin ide:

```
HMW: "Bagaimana kita bisa membantu siswa menemukan starting point yang tepat?"

Ide (tanpa filter dulu):
- Quiz singkat untuk rekomendasi track
- Onboarding wizard 3 langkah
- "Mulai dari sini" berdasarkan background
- Video 60 detik per track sebelum daftar
- Testimonial siswa dengan background serupa
- Progress bar "kamu sudah X% siap untuk track ini"
```

Baru setelah brainstorm, filter berdasarkan feasibility dan impact.

## Latihan

Gunakan hasil interview dari lesson sebelumnya:
1. Buat journey map untuk persona yang sudah kamu buat (bisa di FigJam atau kertas)
2. Identifikasi 3 pain point terbesar
3. Tulis 3 pertanyaan HMW dari pain point tersebut
4. Brainstorm minimal 5 ide per pertanyaan HMW
5. Pilih 1 ide terbaik — ini yang akan kamu wireframe di modul berikutnya
