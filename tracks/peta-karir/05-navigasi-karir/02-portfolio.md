# Membangun Portfolio yang Berbicara

Portfolio bukan kumpulan karya terbaik. Portfolio adalah bukti bahwa kamu bisa menyelesaikan masalah nyata.

## Portfolio vs CV

```
CV berkata:    "Saya bisa melakukan X"
Portfolio membuktikan: "Ini bukti saya melakukan X"

Dunia semakin tidak percaya pada klaim.
Dunia semakin percaya pada bukti.
```

## Apa yang Membuat Portfolio Kuat?

### 1. Proyek Nyata, Bukan Latihan

```
❌ Lemah: "Saya membuat website toko online sebagai tugas sekolah"
✅ Kuat:  "Saya membuat website untuk Warung Bu Sari di Yogyakarta.
           Setelah 3 bulan, mereka mendapat 15 pelanggan baru dari Google Maps."
```

Dampak nyata > kesempurnaan teknis.

### 2. Dokumentasi Proses

```
Untuk setiap proyek, dokumentasikan:
  → Masalah yang diselesaikan
  → Pendekatan yang diambil
  → Tantangan yang dihadapi
  → Hasil yang dicapai
  → Apa yang dipelajari
```

### 3. Konsistensi dan Pertumbuhan

```
Portfolio yang menunjukkan pertumbuhan lebih meyakinkan dari
portfolio yang hanya menampilkan karya terbaik.

Tunjukkan:
  → Proyek pertama (meski sederhana)
  → Proyek terbaru (yang lebih kompleks)
  → Apa yang berubah dan mengapa
```

## Platform Portfolio

### GitHub (untuk Developer)

```
Setiap proyek harus punya README yang menjawab:
  → Apa ini?
  → Mengapa dibuat?
  → Bagaimana cara menjalankannya?
  → Screenshot atau demo
  → Tech stack yang digunakan
  → Apa yang dipelajari
```

### Personal Website

Website pribadi adalah portfolio yang paling fleksibel:

```
Halaman yang wajib ada:
  → Home: siapa kamu dalam 1 kalimat
  → About: cerita yang lebih lengkap
  → Projects: 3-5 proyek terbaik dengan case study
  → Contact: cara menghubungimu

Tools gratis:
  → GitHub Pages + Astro/Hugo (gratis, custom domain)
  → Notion (cepat, tapi kurang custom)
  → Carrd (simple, gratis untuk 1 halaman)
```

## Portfolio untuk Berbagai Tujuan

```
Untuk melamar kerja:
  → Fokus pada proyek yang relevan dengan posisi
  → Tunjukkan kemampuan kolaborasi (kontribusi open source, proyek tim)

Untuk mendapat klien freelance:
  → Fokus pada hasil bisnis (bukan hanya teknis)
  → Testimonial dari klien sebelumnya
  → Tunjukkan proses, bukan hanya hasil akhir

Untuk beasiswa:
  → Tunjukkan dampak sosial
  → Kepemimpinan dan inisiatif
  → Konsistensi dan komitmen jangka panjang
```

## Mulai Sekarang, Sempurnakan Nanti

```
Portfolio yang ada > portfolio yang sempurna tapi belum ada

Langkah minimal hari ini:
  1. Buat GitHub profile dengan README
  2. Upload 1 proyek (meski sederhana)
  3. Tulis README yang jelas

Sempurnakan seiring waktu.
```

## Latihan

1. Audit portfolio-mu saat ini — apa yang ada? Apa yang kurang?
2. Pilih 1 proyek yang sudah kamu kerjakan dan tulis case study singkatnya (200 kata)
3. Buat atau update GitHub profile README
4. Buat personal website sederhana (bisa pakai GitHub Pages + template gratis)
5. Identifikasi: proyek apa yang ingin kamu tambahkan ke portfolio dalam 3 bulan ke depan?
